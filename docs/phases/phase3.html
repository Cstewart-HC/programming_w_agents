Phase 3 10 min

Reviewing Enforcement Mechanisms

Objective: Inspect and understand the local enforcement mechanisms that will catch code quality issues.

Verify Your Configuration

Open pyproject.toml in your editor. You should see something like this:

pyproject.toml

[tool.ruff]
line-length = 88

[tool.ruff.lint]
select = ["E", "W", "F", "C90"]

[tool.ruff.lint.mccabe]
max-complexity = 5
Critical Callout: Notice the max-complexity = 5 rule. We are overriding the industry standard of 15 down to 5 strictly for this workshop so we can force a failure in the next phase.

Do not use 5 in your real projects! Use 15 (or the value your team agrees on).

What These Rules Mean

  • select = ["E", "W", "F", "C90"] - Enables pycodestyle errors/warnings, Pyflakes, and McCabe complexity checks
  • max-complexity = 5 - No function can have cognitive complexity above 5 (workshop only)
  • line-length = 88 - Maximum line width (Black's default)

Pre-Commit Hooks

Open .pre-commit-config.yaml. This file defines hooks that run before every commit:

.pre-commit-config.yaml

repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.3.5
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format

When you run git commit, these hooks will:

  • Run ruff to check for linting errors and auto-fix what it can
  • Run ruff-format to ensure consistent formatting
  • Block the commit if any errors remain
