Phase 2 20 min

Deterministic Setup

Objective: Set up the environment and lock dependencies securely using Agent Mode.

Step 1: The Master Scaffolding Prompt

Open the Copilot Chat Sidebar (Ctrl+Alt+I) and select Agent Mode from the agent selector at the top of the panel. Agent Mode can execute terminal commands and create files simultaneously.

Before you start: Make sure you are not in your home directory or an existing project folder. Check with pwd and navigate to an appropriate location first. Agent Mode will create a dedicated retail_analysis folder for you.
Copy this prompt:
"Act as a senior Python data engineer. Execute these steps sequentially:
  1. Create a new directory called retail_analysis, navigate into it, and initialize a git repository.
  2. Use pip to install the latest versions of pandas, seaborn, jupyter, ruff, and pre-commit.
  3. Run pip freeze > requirements.txt to lock the exact versions.
  4. Scaffold the project directory structure, creating separate folders for data, notebooks, and src.
  5. Generate a standard Python .gitignore file that includes .venv/, __pycache__/, and .env.
  6. Create an empty .github/copilot-instructions.md file.
  7. Create pyproject.toml and .pre-commit-config.yaml files configured to use Ruff. For this demonstration, enforce a strict cognitive complexity limit of 5 (C901) and a maximum line length of 88. Finally, run pre-commit install.
Review the suggested steps and click Run."
Fallback: If Agent Mode fails due to permissions, run these commands manually: 
python -m venv .venv
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # macOS/Linux
pip install pandas seaborn jupyter ruff pre-commit
pip freeze > requirements.txt
pre-commit install

Why this matters: This file acts as both human reference and the AI's executable constraints. Copilot reads this file to understand your coding standards.
