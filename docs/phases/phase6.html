What's Next

Objective: Understand the gaps in today's workflow and what a production-ready data pipeline still needs.

What's Missing?

The Plan-Build-Review loop you learned today is a powerful starting point, but a real data pipeline needs more to be trustworthy:

Unit Tests

How do you know your revenue calculation is correct for edge cases — zero quantities, missing prices, negative returns?

Data Validation

What if the CSV schema changes or a column goes missing? A validation pass would catch that before any analysis runs.

Reproducibility

Are your results the same every run? Without pinned dependencies and deterministic data handling, they might not be.

CI/CD

Right now the hooks run on your machine. In a team setting, they should run automatically on every pull request.

A follow-up session will extend this project with automated testing and data validation, using the same Plan-Build-Review loop with Copilot. You'll learn how to prompt for testable code from the start and use hooks to enforce test coverage — not just style.

Take-Home Challenge

Extend this project by adding a Jupyter notebook that acts as a dashboard for the analysis results.

Dashboard Requirements

  • Read the data from data/retail_sample.csv
  • Display revenue by country as a bar chart
  • Display anomaly counts as a pie chart or summary table
  • Include markdown cells explaining each section
  • Save as notebooks/dashboard.ipynb

Use the same Plan-Build-Review loop to build it — plan the notebook structure first, then build, then review.
