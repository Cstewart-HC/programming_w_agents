Hands-On Workshop: Programming Agents & Enforcing Standards

Audience: Academic & Applied Research, Holland College
Environment: VS Code, GitHub Copilot, Python 3.10+, Git

Phase 0: Preflight Check

5 min
Objective: Verify the environment before we start.

Please ensure:

  1. VS Code is open.
  2. The GitHub Copilot extension is installed and you are signed in (check the icon in the bottom right corner).
  3. You have a terminal open in VS Code (Terminal -> New Terminal).
  4. Python and Git are recognized in your terminal (type python --version and git --version).

Phase 1: The AI Workflow

10 min
Objective: Move from "autocomplete" to an agentic workflow using the Plan-Build-Review loop.

Setup (Pre-planning)

Establish project scaffolding and local enforcement.

The Core Loop (Iterative)

  • Plan: Design the feature implementation in the Copilot Chat Sidebar.
  • Build: Use Copilot Edits (Agent Mode) to implement the planned code across files.
  • Review: Use AI for soft checks, then rely on Git hooks to ruthlessly enforce formatting and complexity limits.

Phase 2: Deterministic Setup

20 min
Objective: Set up the environment and lock dependencies securely.

1The Master Scaffolding Prompt

Open the Copilot Chat Sidebar (Ctrl+Alt+I) and select Agent Mode. Agent Mode can execute terminal commands and create files simultaneously.

"Act as a senior Python data engineer. Execute these steps sequentially:
  1. Initialize a git repository. Create a Python virtual environment named .venv and activate it.
  2. Use pip to install the latest versions of pandas, seaborn, jupyter, ruff, and pre-commit.
  3. Run pip freeze > requirements.txt to lock the exact versions.
  4. Scaffold the project directory structure, creating separate folders for data, notebooks, and src.
  5. Generate a standard Python .gitignore file.
  6. Create an empty .github/copilot-instructions.md file.
  7. Create pyproject.toml and .pre-commit-config.yaml files configured to use Ruff. For this demonstration, enforce a strict cognitive complexity limit of 5 (C901) and a maximum line length of 88. Finally, run pre-commit install.
Review the suggested steps and click Run."
Fallback: If Agent Mode fails due to permissions, you can run these standard commands manually: python -m venv .venv, activate it, pip install pandas ruff pre-commit, pip freeze > requirements.txt, and pre-commit install

2Define Workspace Instructions

Open the .github/copilot-instructions.md file. Paste the Applied Analytics Engineering Principles (from Handout 1) into this file and save it. This acts as both human reference and the AI's executable constraints.

Phase 3: Reviewing Enforcement Mechanisms

10 min
Objective: Inspect the local enforcement.
  • Verify Configuration: Open pyproject.toml.
Critical Callout: Notice the max-complexity = 5 rule. We are overriding the industry standard of 15 down to 5 strictly for this workshop so we can force a failure in the next step. Do not use 5 in your real projects.

Fallback Configs: If the agent hallucinated the config files, replace their contents with the exact code blocks provided in Handout 3.

Phase 4: Feature Development

45 min
Objective: Execute the core development loop using the UCI Online Retail dataset.

1Data Retrieval

Instead of manually downloading files, we will use the agent to fetch the dataset directly from the workshop repository.

Open Agent Mode (Ctrl+Shift+I) and prompt:

"Download the retail_sample.csv file from https://github.com/Cstewart-HC/programming_w_agents/blob/master/data/retail_sample.csv and save it directly into my local data/ directory."

2Plan

In the Copilot Chat Sidebar (Ctrl+Alt+I), map out the approach with strict output definitions:

"Plan the implementation for a script in src/analysis.py that ingests data/retail_sample.csv. Requirements:
  1. Explicitly validate the schema and parse the dates defensively.
  2. Calculate total revenue (Quantity * UnitPrice) per Country.
  3. Flag any returns (negative quantities) or records with missing CustomerIDs as anomalies.
  4. The output must write a report to reports/analysis_report.txt containing a revenue summary by country and an anomaly alert list.
Outline the necessary typed helper functions with docstrings first."

3Build (Agent Mode)

Open Copilot Edits (usually Ctrl+Shift+I). Intentionally ask the agent to build this poorly:

"Implement the plan we just discussed in src/analysis.py. However, do it entirely within a single, highly nested function named process_retail_data ignoring the helper function outline."

4Review (Copilot & Hooks)

Stage A: AI Review (Soft Check)

With src/analysis.py active, open the Copilot Chat Sidebar:

"Review this code. Does it adhere to the complexity rules defined in my workspace instructions?"

(Copilot should warn you, but AI is fallible. We need a hard gate).

Stage B: Hard Enforcement

In your standard terminal, run:

pre-commit run --all-files

Ruff will intercept and explicitly fail the check due to C901 process_retail_data is too complex.

5Iteration / Refactoring

Return to Copilot Edits. Prompt:

"The code failed my pre-commit hook due to high cognitive complexity. Refactor process_retail_data into the smaller, modular helper functions we originally planned so it passes the Ruff linter."

Accept the refactored code. Run pre-commit run --all-files again to verify it passes, then stage and commit your code.
