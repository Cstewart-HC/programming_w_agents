Phase 5 10 min

Iterate

Objective: Run the analysis script, verify the output, and iterate on any issues — this is the manual part of the Review phase in the Plan-Build-Review loop.

1 Run the Script

Your code passes linting — but does it actually work? Open Agent Mode (Ctrl+Shift+I) and prompt:

Copy this prompt:
"Run src/analysis.py and show me the contents of reports/analysis_report.txt."

The script should generate reports/analysis_report.txt containing a revenue summary grouped by country and an anomaly list flagging returns and missing customer IDs.

2 Expected Output

If everything works correctly, your output should look something like this:

=== Revenue by Country === Country Total Revenue Transactions United Kingdom £421,832.15 36,892 Netherlands £17,540.32 437 EIRE £15,823.21 748 Germany £14,327.46 949 France £12,518.73 834 === Anomalies Detected === [!] 1,062 returns (negative quantities) found [!] 1,348 records with missing CustomerID [!] 3,410 total anomalies (8.0% of dataset)

The exact numbers will vary depending on how the agent implemented date parsing and filtering, but you should see a ranked country list and a summary of flagged records.

3 Handle Errors

If the script crashes or produces unexpected output, paste the error directly back to Agent Mode:

Copy this prompt:
"[Paste the full error output here.]"
Don't describe the error in your own words. Paste the raw traceback. The agent can reason about the exact line numbers and types, which leads to faster, more accurate fixes.

After the agent applies a fix, run the script again to confirm. Repeat this cycle until the output looks correct.

Checkpoint: Once the script runs cleanly, use Agent Mode to commit:
Copy this prompt:
"Stage and commit all changes with the message 'Verify analysis output and fix runtime issues'."

Key Takeaways

  • Plan first, build second — The Plan-Build-Review loop prevents wasted effort. Phase 1's plan directly informed Phase 4's build.
  • AI is a soft check, hooks are hard gates — Copilot can suggest improvements, but only automated enforcement catches every issue consistently.
  • Paste errors, don't describe them — Raw tracebacks give the agent the context it needs. Paraphrasing loses critical details.
  • Complexity limits force better design — Small functions aren't just about passing linting; they're easier to debug, test, and maintain.
  • The Review phase is where quality happens — Running the script, inspecting output, and iterating on errors is the manual counterpart to the automated hooks. Both are essential.
