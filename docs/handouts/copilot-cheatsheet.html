Handout 2: Copilot UI & Cheat Sheet

Essential UI Elements

Terminal Chat / Inline Chat

Best for targeted edits or running terminal commands without memorizing syntax.

Open Terminal Chat Ctrl+I

Use when: You need to run commands, create files, or make quick edits without context switching.

Copilot Edits (Agent Mode)

Best for the Build phase when scaffolding or applying changes across multiple files simultaneously.

Open Copilot Edits Ctrl+Shift+I

Use when: Implementing features, refactoring code, or making coordinated changes across multiple files.

Copilot Chat Sidebar

Best for the Plan and Review phases.

Open Chat Sidebar Ctrl+Alt+I

Use when: Designing your approach, asking questions about code, or reviewing implementations.

Context Variables & Commands

@workspace

Reads your entire directory structure and files. Use when you need Copilot to understand the full project context.

#file:<filename>

Type # followed by a filename to force Copilot to read a specific file (e.g., #file:retail_sample.csv).

/review

Analyzes the active file for bugs or style issues.

Reminder: This is a soft check. Your Git hooks are the final authority.

/explain

Explains the selected code or the entire active file in plain language.

/fix

Proposes fixes for problems in the selected code or active file.

/tests

Generates unit tests for the selected code.

Workflow Tips

Effective Prompting

  • Be specific: "Create a function that calculates X" beats "help me with math"
  • Provide context: Use @workspace or #file: to point to relevant files
  • Specify constraints: Mention complexity limits, line length, or style requirements
  • Iterate: If output isn't right, refine your prompt rather than accepting subpar code

The Hard Truth

  • AI suggestions are suggestions - always review before accepting
  • Your pre-commit hooks are the final gate - if they fail, the code doesn't ship
  • Copilot doesn't know your business logic unless you tell it
