Handout 3: Fallback Configurations

If the agent fails to generate the enforcement files correctly during Phase 2, copy and paste these exact blocks.

Important: The max-complexity = 5 setting below is intentionally strict for workshop demonstration purposes. For production projects, use 15 (industry standard) or adjust based on your team's conventions.

pyproject.toml

pyproject.toml
[project]
name = "programming-with-ai-workshop"
version = "0.1.0"
description = "Workshop on programming with AI for Applied Research"
requires-python = ">=3.10"

[tool.ruff]
line-length = 88

[tool.ruff.lint]
select = ["E", "W", "F", "C90"]

[tool.ruff.lint.mccabe]
max-complexity = 5

.pre-commit-config.yaml

.pre-commit-config.yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.3.5
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format

Alternative: Production-Ready Configuration

For real projects after the workshop, use these more practical settings:

pyproject.toml (production)
[project]
name = "your-project-name"
version = "0.1.0"
requires-python = ">=3.10"

[tool.ruff]
line-length = 88

[tool.ruff.lint]
select = ["E", "W", "F", "C90", "PLR"]

[tool.ruff.lint.mccabe]
max-complexity = 15

[tool.ruff.lint.pylint]
max-statements = 50
max-args = 6

Quick Verification Commands

After setting up your configs, verify they work:

# Install pre-commit hooks
pre-commit install

# Run manually on all files
pre-commit run --all-files

# Run only Ruff
ruff check .
ruff format --check .
