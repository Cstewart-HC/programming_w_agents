Phase 2 20 min

Deterministic Setup

Objective: Set up the environment and lock dependencies securely using AI-assisted terminal commands.

Step 1: The Master Scaffolding Prompt

Open the Terminal Chat (press Ctrl+I or Cmd+I while focused in the terminal). This allows Copilot to execute commands directly.

"Act as a senior Python data engineer. Execute these steps sequentially:
  1. Initialize a git repository. Create a Python virtual environment named .venv and activate it.
  2. Use pip to install the latest versions of pandas, seaborn, jupyter, ruff, and pre-commit.
  3. Run pip freeze > requirements.txt to lock the exact versions.
  4. Scaffold the project directory structure, creating separate folders for data, notebooks, and src.
  5. Generate a standard Python .gitignore file.
  6. Create an empty .github/copilot-instructions.md file.
  7. Create pyproject.toml and .pre-commit-config.yaml files configured to use Ruff. For this demonstration, enforce a strict cognitive complexity limit of 5 (C901) and a maximum line length of 88. Finally, run pre-commit install.
Review the suggested steps and click Run."
Fallback: If Terminal Chat fails due to permissions, run these commands manually: 
python -m venv .venv
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # macOS/Linux
pip install pandas seaborn jupyter ruff pre-commit
pip freeze > requirements.txt
pre-commit install

Step 2: Define Workspace Instructions

Open the .github/copilot-instructions.md file. Paste the Applied Analytics Engineering Principles into this file and save it.

Why this matters: This file acts as both human reference and the AI's executable constraints. Copilot reads this file to understand your coding standards.
